You will receive a single project's details in JSON format. Analyze it and return a JSON object with this exact structure:

  "budget_estimate": {

    "max": numeric value in USD (integer),

      { "item": "description of cost item", "amount": numeric USD }
    ]
  },
  "timeline": {
    "total_duration": "e.g. 2 weeks, 1 month",
    "phases": [
      { "phase": "phase name", "duration": "e.g. 3 days", "description": "what happens" }
    ]
  },
  "scaling_prospects": {
    "potential": "low | medium | high",
    "opportunities": ["list of future upsell or expansion opportunities"],
    "risks": ["list of risks that could impact cost or timeline"]
  },
  "risk_assessment": {
    "estimation_confidence": "low | medium | high",
    "scope_creep_risk": "low | medium | high",
    "technical_risk": "low | medium | high",

    "risk_factors": ["specific risks that could cause estimation to be wrong"],

  }
}

Rules:
- Return ONLY a valid JSON object, no explanation or markdown fences
- If budget is provided in the input, use it as an anchor for your estimate
- If no budget is given, estimate based on industry rates for the complexity and category
- Keep phase descriptions concise (1 sentence)
- Opportunities, risks, risk_factors, mitigation_suggestions: max 3 each
- workload_metrics.weeks_at_40hrs = estimated_hours / 40, rounded to 1 decimal place
- overall_risk_score should reflect estimation uncertainty, scope clarity, and technical complexity combined
